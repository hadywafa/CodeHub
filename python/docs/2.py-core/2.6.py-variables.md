## **ğŸ“Œ Understanding `is` vs. `==` in Python**

Python provides two different ways to compare values:

| Operator | **Checks**                                                    | **Example**     |
| -------- | ------------------------------------------------------------- | --------------- |
| `==`     | **Value Equality** (Do the values match?)                     | `5 == 5 â†’ True` |
| `is`     | **Identity Comparison** (Are they the same object in memory?) | `x is y`        |

---

### **ğŸ”¹ Example: `==` vs. `is`**

```python
x = [1, 2, 3]
y = [1, 2, 3]

print(x == y)  # True (Values are the same)
print(x is y)  # False (Different memory locations)
```

âœ… **Lists have different memory addresses even if they contain the same values.**

---

### **ğŸ”¹ Example: `is` with Small Integers & Strings (Interning)**

Python **caches small integers (`-5 to 256`) and short strings**.

```python
a = 100
b = 100
print(a is b)  # True (Both reference the same cached object)

c = "hello"
d = "hello"
print(c is d)  # True (Strings are interned)
```

âœ… **Integers and strings may be stored in the same memory location.**

## **ğŸ“Œ Advanced Concepts: Variable References & Memory Management**

### **ğŸ”¹ How Python Stores Variables in Memory**

- **Variables are references** to objects.
- Python **garbage collects** objects when no references exist.

```python
x = [1, 2, 3]  # List object in memory
y = x          # `y` references the same list
y.append(4)    # Modifies `x` too!
print(x)       # [1, 2, 3, 4]
```

âœ… **Both `x` and `y` reference the same list object.**

---

### **ğŸ”¹ Reference Counting (`sys.getrefcount()`)**

Python keeps track of how many variables reference an object.

```python
import sys

x = [1, 2, 3]
print(sys.getrefcount(x))  # Reference count of `x`
```

âœ… **Objects are garbage collected when the reference count reaches 0.**

### **ğŸ”¹ Global Variable**

âœ… **Global variables can be used inside functions but cannot be modified without `global`.**

### **ğŸ”¹ Modifying a Global Variable Inside a Function**

âœ… **Use `global` keyword to modify global variables inside functions.**

### **ğŸ”¹ `nonlocal` Keyword (For Nested Functions)**


âœ… **`nonlocal` is used in nested functions to modify an outer functionâ€™s variable.**
