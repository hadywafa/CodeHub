## üß† What Is a Constructor?

> A **constructor** is a special method that gets called **automatically** when an object is created using `new`.

It‚Äôs used to:

- üîß Initialize fields or properties
- üßº Ensure valid object state
- üö´ Prevent uninitialized objects

---

## üéØ Syntax of a Constructor

```csharp
public class Person
{
    public string Name;

    // Constructor
    public Person(string name)
    {
        Name = name;
    }
}
```

Usage:

```csharp
var p = new Person("Hady");
Console.WriteLine(p.Name); // üëâ "Hady"
```

> üîê It **has the same name** as the class
> üö´ It **doesn't return anything** (not even `void`)

## üß± 1. Default Constructor

A constructor with **no parameters**.

```csharp
public class Car
{
    public string Model = "Unknown";

    public Car()  // Default constructor
    {
        Console.WriteLine("Default constructor called!");
    }
}
```

Usage:

```csharp
var c = new Car(); // Auto-calls the default constructor
```

> üß† If you don‚Äôt define any constructor, the compiler **automatically adds** a default one.

‚ö†Ô∏è But if you **add any other constructor**, the default one **is not auto-added anymore**!

---

## üì¶ 2. Parameterized Constructor

Used to pass values at the moment of creation.

```csharp
public class Car
{
    public string Model;

    public Car(string model)
    {
        Model = model;
    }
}
```

Usage:

```csharp
var c = new Car("BMW");
Console.WriteLine(c.Model); // üëâ "BMW"
```

> ‚úÖ Great for **forcing valid initial state**

---

## üîÅ 3. Constructor Overloading

You can create **multiple constructors** with different parameter lists:

```csharp
public class Car
{
    public string Model;
    public int Year;

    public Car() {} // Default

    public Car(string model)
    {
        Model = model;
    }

    public Car(string model, int year)
    {
        Model = model;
        Year = year;
    }
}
```

> üß† C# picks the constructor based on **arguments you pass**

---

## ‚ö° 4. Static Constructor

> A special constructor that runs **once per type**, before any instance or static member is used.

```csharp
public class Logger
{
    static Logger()
    {
        Console.WriteLine("Static constructor runs once!");
    }

    public static void Log(string msg)
    {
        Console.WriteLine(msg);
    }
}
```

Usage:

```csharp
Logger.Log("Hello");  // üëâ Static constructor runs here
```

### ‚úÖ Static Constructor Rules

| Rule                | Explanation                     |
| ------------------- | ------------------------------- |
| No parameters       | Can't pass values to it         |
| Called once         | Per type, before anything else  |
| Can't call manually | Runs automatically              |
| Usually used for    | Initializing static data/config |

## üß¨ 5. Copy Constructor (Manually Defined)

> A **copy constructor** creates a new object by **copying the values** from another object of the same type.

üîß Not built-in automatically like in C++, but you can define it manually:

```csharp
public class Person
{
    public string Name;
    public int Age;

    // Copy Constructor
    public Person(Person other)
    {
        Name = other.Name;
        Age = other.Age;
    }
}
```

### Usage

```csharp
var original = new Person { Name = "Hady", Age = 30 };
var copy = new Person(original);

Console.WriteLine(copy.Name); // üëâ "Hady"
```

### üß† Why Use It?

- Clone objects manually (especially for **value types inside reference objects**)
- Avoid unwanted shared references (shallow vs deep copy)
- Helpful in **immutable object patterns**

---

## üîí 6. Private Constructor

> A **private constructor** is one you define with the `private` access modifier, so it **cannot be called** from outside the class.

```csharp
public class Singleton
{
    public static readonly Singleton Instance = new Singleton();

    // Private constructor
    private Singleton()
    {
        Console.WriteLine("Singleton instance created.");
    }
}
```

### Usage

```csharp
var s = Singleton.Instance;  // ‚úÖ Only way to get it
```

### üß† Use Cases

| Use Case             | Why Private Constructor?              |
| -------------------- | ------------------------------------- |
| ‚úÖ Singleton pattern | Only one instance is ever created     |
| ‚úÖ Static class      | Prevent instantiation                 |
| ‚úÖ Factory pattern   | Force creation through factory method |
| ‚úÖ Prevent misuse    | Hide constructor for API safety       |

## üß∞ Constructor Chaining (this)

You can reuse another constructor from within one:

```csharp
public class Book
{
    public string Title;
    public int Pages;

    public Book(string title) : this(title, 0) {}

    public Book(string title, int pages)
    {
        Title = title;
        Pages = pages;
    }
}
```

> ‚úÖ `this(...)` calls another constructor in same class
> üöÄ Helps **avoid duplication**

## üö´ Common Mistakes to Avoid

| Mistake                                                              | Why it‚Äôs wrong                      |
| -------------------------------------------------------------------- | ----------------------------------- |
| Forgetting to add default constructor when adding parameterized ones | Compiler won't add it automatically |
| Using `void` return on constructor                                   | Constructor has no return type      |
| Trying to call static constructor manually                           | Not allowed                         |
| Doing too much work in constructors                                  | Keep them lean and focused          |
