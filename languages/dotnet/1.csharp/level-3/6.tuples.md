## üß† What Are Tuples?

> A **tuple** is a lightweight, immutable **data container** that groups multiple values into a **single object** ‚Äî without needing a custom class or struct.

---

### üß± 1. Creating Tuples

```csharp
var user = ("Hady", 29);
Console.WriteLine(user.Item1); // "Hady"
Console.WriteLine(user.Item2); // 29
```

‚úÖ Tuple is created using parentheses `()`
‚úÖ Members are named `Item1`, `Item2`, etc. (by default)

---

### üß± 2. Naming Tuple Elements

```csharp
var user = (Name: "Hady", Age: 29);

Console.WriteLine(user.Name); // üëâ "Hady"
Console.WriteLine(user.Age);  // üëâ 29
```

‚úÖ Named tuples make code more readable and self-documenting

---

### üß∞ 3. Returning Tuples from Methods

```csharp
(string FirstName, string LastName) GetUser()
{
    return ("Hady", "Wafa");
}
```

Usage:

```csharp
var user = GetUser();
Console.WriteLine(user.FirstName); // Hady
```

---

### ü™Ñ 4. Deconstruction

> Deconstructing a tuple means **unpacking** it into individual variables

```csharp
var (first, last) = GetUser();
Console.WriteLine($"{first} {last}"); // Hady Wafa
```

‚úÖ Cleaner than accessing `.Item1`, `.Item2`, or `.Name`, `.Age`

---

## üß± 5. Deconstruct Custom Classes (Bonus)

You can also enable deconstruction for your own classes:

```csharp
public class Point
{
    public int X { get; }
    public int Y { get; }

    public Point(int x, int y) => (X, Y) = (x, y);

    public void Deconstruct(out int x, out int y)
    {
        x = X;
        y = Y;
    }
}
```

Usage:

```csharp
var point = new Point(3, 4);
var (x, y) = point;
Console.WriteLine($"{x}, {y}"); // üëâ 3, 4
```

## ‚úÖ When to Use Tuples?

| Use Case                      | Why Use Tuples?                    |
| ----------------------------- | ---------------------------------- |
| Return multiple values        | Without creating a class/struct    |
| Quick groupings               | Temporary data combos (e.g., LINQ) |
| Lightweight projections       | Anonymous-like behavior            |
| Destructuring for readability | Named unpacking in one line        |
