## üß© 1. `var` ‚Äì Implicit Typing (Compile-Time Safe)

> `var` lets the **compiler infer the type**, but the type is still **strongly typed**.

```csharp
var name = "Hady";       // string
var age = 29;            // int
var user = new User();   // User
```

### ‚úÖ Pros

- Cleaner syntax, especially for anonymous types or LINQ
- Type is still **resolved at compile-time**
- Prevents redundancy (`Dictionary<string, List<int>>`)

### ‚ùå Misconception

```csharp

```

> **`var` is NOT dynamic** ‚Äî it‚Äôs just _type inference_ at compile time.

## üß™ 2. `dynamic` ‚Äì Runtime Typing

> A `dynamic` variable **defers type checking until runtime**.

```csharp

x = 5; // ‚úÖ No compile-time error
Console.WriteLine(x + 10); // 15
```

### ‚úÖ Pros

- Interop with COM, Office, or JSON
- Works with unknown or changing types
- Great with reflection, duck typing

### ‚ùå Cons

- No IntelliSense
- No compile-time checking (risk of runtime errors!)
- Slower than statically typed code

```csharp
dynamic d = "test";
Console.WriteLine(d.Foo()); // üí• Runtime error if Foo doesn‚Äôt exist
```

## üß¨ 3. Reflection ‚Äì Inspecting Metadata at Runtime

> Reflection lets you **inspect and interact with types at runtime**, including:

- Properties
- Methods
- Attributes
- Assemblies
- Even creating objects dynamically!

---

### üß± Basic Reflection Example

```csharp
Type type = typeof(User);
Console.WriteLine(type.Name);  // üëâ "User"

var properties = type.GetProperties();
foreach (var prop in properties)
    Console.WriteLine(prop.Name);
```

---

### üõ† Use Case: Dynamic Property Access

```csharp
object obj = new User { Name = "Hady" };
Type t = obj.GetType();
PropertyInfo p = t.GetProperty("Name");

Console.WriteLine(p.GetValue(obj)); // üëâ "Hady"
```

---

### üì¶ Use Case: Plugin/Assembly Loading

```csharp

Type pluginType = asm.GetType("MyPlugin.Main");

object instance = Activator.CreateInstance(pluginType);
MethodInfo method = pluginType.GetMethod("Execute");
method.Invoke(instance, null);
```

‚úÖ Enables:

- Plugin systems
- Late binding
- Inspecting types you don‚Äôt know at compile-time

## üí¨ Interview Insight

> ‚ÄúUse `var` for clean syntax when the type is obvious. Use `dynamic` only when type flexibility is truly needed. Use Reflection when you must inspect types at runtime, like for plugin loading, serializers, or frameworks.‚Äù
