# File Path in Python

## The Problem: Why `FileNotFoundError` Happens

Imagine you have a file structure like this in your project:

```text
my_workspace/
│
├── code/
│   └── analysis.py
└── my_project/
    └── data.txt

```

In your code (`analysis.py`), you might naturally try to open the file like this:

```python
# analysis.py
open("data.txt", "r")

```

Even though `data.txt` exists inside your workspace, Python throws a `FileNotFoundError`. Why? Because `data.txt` is only a fragment of the complete location, and Python doesn't know where to start looking.

---

### Solution 1: Absolute Paths (The Quick but Brittle Fix)

An **Absolute Path** specifies the complete route to a file starting all the way from the root of your file system.

* **Windows:** Starts with a drive letter (e.g., `C:\`) and uses backslashes.
* **Mac/Linux:** Starts with a forward slash `/`

#### Code Example (Windows):

```python
# We use an 'r' before the string to make it a "raw string" so Python doesn't misinterpret backslashes
file_path = r"C:\Users\YourName\Documents\my_workspace\my_project\data.txt"

with open(file_path, "r") as file:
    content = file.read()
    print(content)

```

#### Why you shouldn't use it:

Absolute paths are **hardcoded** to your exact machine. If you share this script with a colleague, the code will instantly break on their machine because their username or folder structure will be completely different.

---

### Solution 2: Relative Paths & The Current Working Directory (CWD)

A **Relative Path** specifies only part of the full address, depending on a starting point.

When Python evaluates a relative path, it combines it with your **Current Working Directory (CWD)**—the folder from which Python is executing your script.

You can check your CWD using Python's built-in `os` module:

```python
import os

print("Current Working Directory is:", os.getcwd())

```

If your code editor (like VS Code) runs your script from the `code/` folder, your CWD is `.../my_workspace/code/`.

* If you look for `data.txt`, Python looks *directly* inside `code/` and fails.
* To make it work as a relative path, you have to guide Python from that CWD:

```python
# If CWD is 'code/', we have to tell it how to reach the project folder (this varies based on where you run it)
# A common terminal fix is to manually change directory (cd) to the right folder first:
# Terminal command: cd my_project
# Terminal command: python analysis.py

with open("data.txt", "r") as file:
    print(file.read())

```

#### Why you shouldn't rely on it:

It's easy to forget to switch terminal folders, and relying on the user to run the code from the exact right directory makes the script unreliable.

---

### Solution 3: The Robust Solution using `__file__` (Best Practice)

To write code that works on *any* machine, no matter *where* or *how* it's run, you need a dynamic anchor point. Python provides a built-in variable called `__file__` that always holds the full absolute path of the script currently running.

Using the `os` module, we can extract the parent folder of the script and reliably attach our target file's name.

#### Code Example:

```python
import os

# 1. Look at the path of the current script
print("This script is located at:", __file__) # [00:07:24]

# 2. Get the directory name (parent folder) where this script lives
script_dir = os.path.dirname(__file__) # [00:07:56]
print("The parent directory is:", script_dir)

# 3. Safely join the script directory to the relative location of the data file
# os.path.join handles both Windows (\) and Mac/Linux (/) slashes automatically
data_path = os.path.join(script_dir, "data.txt") # [00:08:36]

# 4. Open and read the file safely
with open(data_path, "r") as file:
    content = file.read()
    print(content)

```

## Summary of Benefits

By switching from hardcoded absolute paths to anchoring your pathing logic via `__file__`:

1. **Portability:** Your code works perfectly if shared with a friend
2. **Execution Agnostic:** It will successfully run whether you click a "Run" button in your IDE or trigger it via the terminal from an entirely different folder

For further deep-dives into file pathing, Python also features a modern, object-oriented approach via the `pathlib` module
