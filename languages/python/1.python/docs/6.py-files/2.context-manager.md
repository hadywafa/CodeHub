## What Are Context Managers?

Context managers in Python are used to manage resources like:

- files or
- database connections

by ensuring they are properly set up and torn down.  
They use the `with` keyword to wrap blocks of code, handling cleanup automatically—even if an error occurs.

> **Key Benefit:** They replace manual management (like `.close()`), preventing memory leaks and data corruption.

---

## 1. Basic File Handling

Using the `with` statement ensures that a file is automatically closed after the block of code finishes, even if an exception is raised.

**Example:**

```python
# Safe file writing
with open('books.txt', 'w') as file:
    file.write('The Great Gatsby\n')
    file.write('The Hunger Games\n')
# The file is automatically closed here.
```

## 2. Managing Multiple Resources

If you need to read from one file and write to another, you can handle both connections simultaneously on a single `with` line to avoid excessive nesting.

**Example:**

```python
# Flat structure for multiple files
with open('books.txt', 'r') as input_file, open('catalog.txt', 'w') as output_file:
    data = input_file.read()
    output_file.write(data)
```

## 3. Database Connections

Context managers are essential for databases to ensure that transactions are committed or rolled back properly. The video demonstrates this using `sqlite3`.

**Example:**

```python
import sqlite3

# Managing a database connection safely
with sqlite3.connect('books.db') as connection:
    cursor = connection.cursor()
    cursor.execute('CREATE TABLE IF NOT EXISTS books (id INTEGER, title TEXT)')
    cursor.execute('INSERT INTO books VALUES (?, ?)', (1, 'The Great Gatsby'))
# Changes are committed and the connection is closed automatically.
```

## Summary of Best Practices

- **Always use `with`** for files and network/database connections.
- **Avoid nesting:** Use comma-separated `with` statements when handling multiple resources.
- **Safety:** Relying on context managers creates more robust programs by ensuring resource cleanup regardless of execution success (02:19-04:23).
