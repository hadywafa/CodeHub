# **ğŸš€ Python Object-Oriented Programming (OOP): Classes & Objects - Advanced Guide**

Python is a fully **object-oriented language**, where everything is an **object**. Understanding **classes and objects** is key to writing clean, reusable, and maintainable code. This guide covers **class structures, object creation, attributes, methods, memory management, and advanced OOP concepts**.

> **Note for C# Developers:** Python has **no explicit access modifiers (`public`, `private`, `protected`)** like C#. Also, Python does **not require `new` when creating objects**.

---

## **ğŸ“Œ 1. What is a Class and an Object?**

### **ğŸ”¹ Class: Blueprint for Objects**

A **class** is a template or blueprint for creating objects.

```python
class Car:
    pass  # Empty class (Can be defined later)
```

âœ… In Python, **`class`** defines a new class.

---

### **ğŸ”¹ Object: Instance of a Class**

An **object** is an instance of a class, created from its blueprint.

```python
my_car = Car()  # Creating an instance of Car
print(my_car)   # Output: <__main__.Car object at 0x...>
```

âœ… In Python, **objects are dynamically created at runtime**. No need for `new` as in C#.

---

## **ğŸ“Œ 2. The `__init__` Constructor (Equivalent to C# Constructor)**

Python uses the `__init__` method as a constructor to **initialize object attributes**.

```python
class Car:
    def __init__(self, brand, color):
        self.brand = brand
        self.color = color

car1 = Car("Toyota", "Red")
print(car1.brand)  # Toyota
```

âœ… `__init__` is equivalent to **C# constructors**.  
âœ… `self` is like `this` in C# and refers to the current instance.

---

## **ğŸ“Œ 3. Instance Attributes vs. Class Attributes**

### **ğŸ”¹ Instance Attributes (Unique per Object)**

Each object gets its **own copy** of instance attributes.

```python
class Car:
    def __init__(self, brand):
        self.brand = brand  # Instance attribute

car1 = Car("BMW")
car2 = Car("Tesla")

print(car1.brand)  # BMW
print(car2.brand)  # Tesla
```

âœ… **Instance attributes are unique to each object**, unlike class attributes.

---

### **ğŸ”¹ Class Attributes (Shared Across All Objects)**

Class attributes are **shared by all instances**.

```python
class Car:
    wheels = 4  # Class attribute

car1 = Car()
car2 = Car()

print(car1.wheels)  # 4
print(car2.wheels)  # 4
```

âœ… **Python uses class attributes like `static` attributes in C#**.

---

## **ğŸ“Œ 4. Methods in Python Classes**

### **ğŸ”¹ Instance Methods (Work on Individual Objects)**

Instance methods **require `self`** and work on a specific object.

```python
class Car:
    def __init__(self, brand):
        self.brand = brand

    def drive(self):
        print(f"{self.brand} is driving!")

car1 = Car("Audi")
car1.drive()  # Audi is driving!
```

âœ… Equivalent to **C# instance methods using `this`**.

---

### **ğŸ”¹ Class Methods (`@classmethod`)**

Class methods **operate on the class itself, not objects**.

```python
class Car:
    wheels = 4  # Class attribute

    @classmethod
    def change_wheels(cls, new_wheels):
        cls.wheels = new_wheels

Car.change_wheels(6)
print(Car.wheels)  # 6
```

âœ… Equivalent to **C# `static` methods modifying `static` attributes**.

---

### **ğŸ”¹ Static Methods (`@staticmethod`)**

Static methods **do not access instance or class attributes**.

```python
class Car:
    @staticmethod
    def honk():
        print("Beep Beep!")

Car.honk()  # Beep Beep!
```

âœ… Equivalent to **C# `static` methods** that don't modify instance/class attributes.

---

## **ğŸ“Œ 5. Memory Management & Object Identity**

### **ğŸ”¹ Checking Object Identity (`id()`)**

Each object has a **unique memory location**.

```python
car1 = Car("Toyota")
car2 = Car("Toyota")

print(id(car1))  # Different memory addresses
print(id(car2))
```

âœ… Equivalent to **C# `ReferenceEquals()`**.

---

### **ğŸ”¹ Object Deletion (`del`)**

Python **automatically manages memory**, but you can manually delete objects.

```python
car1 = Car("Toyota")
del car1  # Deletes the object
```

âœ… **Garbage collection removes unused objects automatically**, similar to C#'s **GC (Garbage Collector)**.

## **ğŸ“Œ 6. Special Methods in Python OOP**

### **ğŸ”¹ `__str__` (User-Friendly Representation)**

```python
class Car:
    def __init__(self, brand):
        self.brand = brand

    def __str__(self):
        return f"Car: {self.brand}"

car = Car("Tesla")
print(car)  # Car: Tesla
```

âœ… Equivalent to **C# `ToString()` method**.

---

### **ğŸ”¹ `__repr__` (Developer-Friendly Representation)**

```python
class Car:
    def __repr__(self):
        return f"Car('{self.brand}')"

car = Car("Tesla")
print(repr(car))  # Car('Tesla')
```

âœ… Equivalent to **C#'s `ToString()` for debugging purposes**.

---

## **ğŸ“Œ 7. Encapsulation: Private & Protected Attributes**

Python **does not have explicit access modifiers** (`public`, `private`, `protected`). Instead, conventions are used.

### **ğŸ”¹ Private Attributes (`__var`)**

```python
class Car:
    def __init__(self, brand):
        self.__brand = brand  # Private variable (Name Mangling)

    def get_brand(self):
        return self.__brand

car = Car("Tesla")
# print(car.__brand)  # âŒ AttributeError
print(car.get_brand())  # âœ… Tesla
```

âœ… **Equivalent to C# `private` variables**.

---

### **ğŸ”¹ Protected Attributes (`_var`)**

```python
class Car:
    def __init__(self, brand):
        self._brand = brand  # Protected (Convention only)

car = Car("BMW")
print(car._brand)  # ğŸš¨ Works, but not recommended (for internal use)
```

âœ… **Equivalent to C# `protected` but without enforcement**.
