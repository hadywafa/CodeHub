## üñ•Ô∏è **Recommended Python Setup for Windows**

### ‚úÖ **1. Install Python for User (Not System-Wide)**

Avoid system-wide installation to prevent conflicts.

#### **Step 1: Download Python**

üëâ Download from: [https://www.python.org/downloads/](https://www.python.org/downloads/)

#### **Step 2: Install with These Options:**

‚úÖ **Check** `"Add Python to PATH"`  
‚úÖ **Select** `"Install for my user only"` (Avoid system-wide installation)

#### **Step 3: Verify Installation**

```powershell
py -0
```

Example output:

```
 -V:3.13 *        Python 3.13 (64-bit)
 -V:3.12          Python 3.12 (64-bit)
```

üëâ `py` (Python Launcher) helps manage versions easily.

---

### ‚úÖ **2. Managing Multiple Python Versions in Windows**

#### **Check Installed Versions**

```powershell
py -0
```

#### **Run a Specific Python Version**

```powershell
py -3.13
```

#### **Set Python 3.13 as Default**

1Ô∏è‚É£ Open the `py.ini` config file:

```powershell
notepad $env:LOCALAPPDATA\py.ini
```

2Ô∏è‚É£ Add:

```
[defaults]
python=3.13
```

üëâ **Now, `py` will always default to Python 3.13.**

---

### ‚úÖ **3. Set Up a Global Virtual Environment (`global-venv`)**

Since you need Jupyter and IPython **globally available**, we use a dedicated virtual environment.

#### **Step 1: Create `global-venv`**

```powershell
py -m venv C:\Users\YourUser\global-venv
```

#### **Step 2: Activate `global-venv`**

```powershell
C:\Users\YourUser\global-venv\Scripts\Activate
```

#### **Step 3: Install Jupyter & IPython**

```powershell
pip install ipython jupyter
```

#### **Step 4: Add Shortcut for Easy Activation**

1Ô∏è‚É£ Open PowerShell profile:

```powershell
notepad $PROFILE
```

2Ô∏è‚É£ Add this line:

```powershell
function globalvenv { & C:\Users\YourUser\global-venv\Scripts\Activate }
```

3Ô∏è‚É£ Save and restart PowerShell.

üëâ Now, simply type:

```powershell
globalvenv
```

To activate your environment.

#### **Step 5: Run Jupyter Notebook**

```powershell
jupyter notebook
```

‚úÖ **Now, IPython & Jupyter are available globally, while keeping system Python clean.**

---

### ‚úÖ **4. (Optional) Use WSL for a Linux-Like Environment**

If you frequently work on **Linux-based deployments**, install **Windows Subsystem for Linux (WSL)**:

```powershell
wsl --install
```

Then install Python inside WSL:

```bash

```

üëâ **This allows you to develop in a Linux-like environment on Windows.**

## üêß **Recommended Python Setup for Linux**

### ‚úÖ **1. Install Python System-Wide for System Tools**

```bash
sudo apt update
sudo apt install python3 python3-pip python3-venv
```

üëâ **This installs Python in `/usr/bin/python3` for system utilities.**

---

### ‚úÖ **2. Install Python Per User for Development**

To prevent system conflicts, install Python only for your user.

#### **Step 1: Install Python 3.13 for Your User**

```bash
mkdir -p ~/.local/bin
cd /tmp
curl -O https://www.python.org/ftp/python/3.13.0/Python-3.13.0.tar.xz
tar -xf Python-3.13.0.tar.xz
cd Python-3.13.0
./configure --prefix=$HOME/.local --enable-optimizations
make -j$(nproc)
make install
```

#### **Step 2: Add User Python to `$PATH`**

```bash
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
```

üëâ **Now, `python3` defaults to your user-installed version.**

---

### ‚úÖ **3. Set Up a Global Virtual Environment (`global-venv`)**

To keep Jupyter & IPython globally available **but isolated**, use a dedicated virtual environment.

#### **Step 1: Create `global-venv`**

```bash
python3 -m venv ~/global-venv
```

#### **Step 2: Activate `global-venv`**

```bash
source ~/global-venv/bin/activate
```

#### **Step 3: Install Jupyter & IPython**

```bash
pip install ipython jupyter
```

#### **Step 4: Add Shortcut for Easy Activation**

```bash
echo 'alias globalvenv="source ~/global-venv/bin/activate"' >> ~/.bashrc
source ~/.bashrc
```

Now, simply type:

```bash
globalvenv
```

To activate your environment.

#### **Step 5: Run Jupyter Notebook**

```bash
jupyter notebook
```

‚úÖ **Now, Jupyter Notebook & IPython are globally available while keeping system Python clean.**
