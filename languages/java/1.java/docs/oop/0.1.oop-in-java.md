# üß± OOP in Java (for C# Developers)

> ‚úÖ Java is a **purely Object-Oriented** programming language (except for a few primitives like `int`, `boolean` etc.), and it follows the **4 fundamental OOP principles** just like C#:

- **Encapsulation**
- **Abstraction**
- **Inheritance**
- **Polymorphism**

Let‚Äôs break down each one ‚Äî with Java syntax and C# comparisons.

---

## üîê 1. Encapsulation

> Hiding internal state and exposing only necessary functionality (via getters/setters)

### ‚úÖ Java Example:

```java
public class BankAccount {
    private double balance;

    public void deposit(double amount) {
        balance += amount;
    }

    public double getBalance() {
        return balance;
    }
}
```

### ü§ñ Compared to C#:

```csharp
public class BankAccount {
    private double balance;

    public void Deposit(double amount) {
        balance += amount;
    }

    public double GetBalance() {
        return balance;
    }
}
```

‚úÖ In both languages:

- Fields are `private`
- Methods control access
- Can use properties (Java 21 now has **record** and **data classes** like C#'s auto-properties)

---

## üßΩ 2. Abstraction

> Hiding implementation details behind interfaces or abstract classes

### ‚úÖ Java Example:

```java
abstract class Animal {
    abstract void makeSound();
}

class Dog extends Animal {
    void makeSound() {
        System.out.println("Woof!");
    }
}
```

### ü§ñ Compared to C#:

```csharp
abstract class Animal {
    public abstract void MakeSound();
}

class Dog : Animal {
    public override void MakeSound() {
        Console.WriteLine("Woof!");
    }
}
```

üß† Both use `abstract` classes and methods. Java also supports interfaces with `default` methods like C#'s interface implementations.

---

## üß¨ 3. Inheritance

> Reusing existing class behavior using `extends` (Java) or `:` (C#)

### ‚úÖ Java Example:

```java
class Vehicle {
    void start() {
        System.out.println("Vehicle started");
    }
}

class Car extends Vehicle {
    void honk() {
        System.out.println("Beep beep!");
    }
}
```

### ü§ñ Compared to C#:

```csharp
class Vehicle {
    public void Start() => Console.WriteLine("Vehicle started");
}

class Car : Vehicle {
    public void Honk() => Console.WriteLine("Beep beep!");
}
```

‚úÖ Key difference:

- Java uses `extends`, not `:`
- Java doesn‚Äôt support **multiple inheritance** (like C#), but both support interfaces.

---

## ü¶æ 4. Polymorphism

> Same method name, different behavior depending on the object (runtime and compile-time)

### ‚úÖ Java Runtime Polymorphism:

```java
class Animal {
    void makeSound() {
        System.out.println("Some sound");
    }
}

class Cat extends Animal {
    void makeSound() {
        System.out.println("Meow");
    }
}

Animal a = new Cat();
a.makeSound();  // Meow
```

### ü§ñ C# Version:

```csharp
class Animal {
    public virtual void MakeSound() => Console.WriteLine("Some sound");
}

class Cat : Animal {
    public override void MakeSound() => Console.WriteLine("Meow");
}
```

üß† Java doesn't need `virtual`/`override` keywords by default.
It uses **method overriding** automatically if names and signatures match (but `@Override` is recommended for clarity).

## üé≠ Java vs C# OOP Summary Table

| Concept                 | Java                                                        | C#                                           |
| ----------------------- | ----------------------------------------------------------- | -------------------------------------------- |
| Inheritance             | `extends`, single only                                      | `:`, supports multiple via interfaces        |
| Interface               | `interface`, `default` methods allowed (Java 8+)            | `interface`, default impls (C# 8+)           |
| Access Modifiers        | `public`, `private`, `protected`, package-private (default) | `public`, `private`, `protected`, `internal` |
| Constructor Overloading | ‚úÖ Yes                                                      | ‚úÖ Yes                                       |
| Operator Overloading    | ‚ùå Not supported                                            | ‚úÖ Supported                                 |
| Properties              | Manual getter/setter methods                                | Auto-properties available                    |
| Abstract Class          | `abstract class`, `abstract` methods                        | Same concept                                 |
| Final Keyword           | `final` (like C#'s `sealed` or `readonly`)                  | `sealed`, `const`, `readonly`                |

## üß™ Bonus ‚Äì Class Template in Java

```java
public class Person {
    private String name;

    // Constructor
    public Person(String name) {
        this.name = name;
    }

    // Getter
    public String getName() {
        return name;
    }

    // Setter
    public void setName(String name) {
        this.name = name;
    }

    // Behavior
    public void sayHello() {
        System.out.println("Hello, " + name);
    }
}
```

Just like in C#, you define:

- Fields (`private`)
- Methods (`public`)
- Constructors
- Accessors (get/set)
