# **ğŸš€ Python Abstract Classes & Interfaces (`ABC` Module) - A Complete Guide**

In Python, **abstract classes** and **interfaces** provide a blueprint for other classes, ensuring that subclasses implement specific methods. Python uses the **`abc` (Abstract Base Class) module** to create **abstract classes and enforce method implementation** in subclasses.

---

## **ğŸ“Œ 1. What are Abstract Classes and Interfaces?**

| **Concept**         | **Description**                                                                           |
| ------------------- | ----------------------------------------------------------------------------------------- |
| **Abstract Class**  | A class that **cannot be instantiated** and contains one or more **abstract methods**.    |
| **Abstract Method** | A method **without implementation** that must be overridden in subclasses.                |
| **Interface**       | A class that defines only method signatures, ensuring subclasses implement those methods. |

> **C# Note**: In **C#**, `abstract class` and `interface` are distinct concepts. In Python, **interfaces are implemented using abstract classes (`ABC` module)**.

## **ğŸ“Œ 2. The `abc` (Abstract Base Class) Module**

Python provides the **`abc` (Abstract Base Class) module**, which allows us to **define abstract classes and enforce implementation** in subclasses.

### **ğŸ”¹ Importing `ABC` and `abstractmethod`**

## **ğŸ“Œ 3. Creating an Abstract Class**

An **abstract class** contains at least **one abstract method** and **cannot be instantiated**.

```python
from abc import ABC, abstractmethod

class Animal(ABC):  # Abstract Base Class
    @abstractmethod
    def make_sound(self):
        pass  # No implementation

# Trying to instantiate will raise an error
# animal = Animal()  # âŒ TypeError: Can't instantiate abstract class
```

âœ… **Abstract classes cannot be instantiated directly**.  
âœ… **Subclasses must implement abstract methods**.

---

## **ğŸ“Œ 4. Implementing an Abstract Class**

A subclass **must override** all abstract methods of its parent abstract class.

### **ğŸ”¹ Example: Implementing Abstract Methods**

```python
class Dog(Animal):
    def make_sound(self):
        return "Woof!"

class Cat(Animal):
    def make_sound(self):
        return "Meow!"

dog = Dog()
cat = Cat()

print(dog.make_sound())  # âœ… Woof!
print(cat.make_sound())  # âœ… Meow!
```

âœ… **All abstract methods must be implemented** in the subclass.

---

## **ğŸ“Œ 5. Enforcing Method Implementation**

If a subclass **does not implement** all abstract methods, it **cannot be instantiated**.

### **ğŸ”¹ Example: Missing Implementation Causes Error**


âœ… **Python ensures all abstract methods are implemented before allowing object creation.**

## **ğŸ“Œ 6. Abstract Classes with Concrete Methods**

An **abstract class can have concrete (fully implemented) methods**.

### **ğŸ”¹ Example: Abstract Class with Concrete Method**

```python
class Vehicle(ABC):
    @abstractmethod
    def start_engine(self):
        pass  # Abstract method

    def stop_engine(self):  # Concrete method
        print("Engine stopped.")

class Car(Vehicle):
    def start_engine(self):
        print("Car engine started.")

car = Car()
car.start_engine()  # âœ… Car engine started.
car.stop_engine()  # âœ… Engine stopped.
```

âœ… **Concrete methods allow code reuse while enforcing required methods.**

## **ğŸ“Œ 8. Creating an Interface (Abstract Class with Only Abstract Methods)**

A **pure interface** contains only **abstract methods**.

### **ğŸ”¹ Example: Defining an Interface**

```python
class Flyable(ABC):
    @abstractmethod
    def fly(self):
        pass

class Bird(Flyable):
    def fly(self):
        print("Bird is flying.")

bird = Bird()
bird.fly()  # âœ… Bird is flying.
```

> **C# Note:** Python **does not have a separate `interface` keyword** like C#. Instead, interfaces are created using **abstract classes with only abstract methods**.

## **ğŸ“Œ 9. Implementing Multiple Interfaces (Multiple Inheritance)**

Python allows **multiple inheritance**, so a class can implement multiple interfaces.

### **ğŸ”¹ Example: Implementing Multiple Interfaces**

```python
class Flyable(ABC):
    @abstractmethod
    def fly(self):
        pass

class Swimmable(ABC):
    @abstractmethod
    def swim(self):
        pass

class Duck(Flyable, Swimmable):
    def fly(self):
        print("Duck is flying.")

    def swim(self):
        print("Duck is swimming.")

duck = Duck()
duck.fly()  # âœ… Duck is flying.
duck.swim()  # âœ… Duck is swimming.
```

âœ… **Python allows implementing multiple interfaces directly, unlike C# which uses explicit interface implementation.**

## **ğŸ“Œ 10. Using Abstract Classes for Dependency Injection**

Abstract classes are useful for **dependency injection** and **polymorphism**.

### **ğŸ”¹ Example: Dependency Injection with Abstract Class**

```python
class Logger(ABC):
    @abstractmethod
    def log(self, message):
        pass

class ConsoleLogger(Logger):
    def log(self, message):
        print(f"LOG: {message}")

class Application:
    def __init__(self, logger: Logger):
        self.logger = logger

    def run(self):
        self.logger.log("Application started.")

app = Application(ConsoleLogger())
app.run()  # âœ… LOG: Application started.
```

âœ… **Abstract classes enable flexibility in choosing different implementations at runtime.**
