## üß† What is an Exception?

An **exception** is a **runtime error** ‚Äî something unexpected that breaks normal program flow.
It could be:

- A `DivideByZeroException`
- A `NullReferenceException`
- A failed API call
- A missing file
- Or‚Ä¶ a typo like `int.Parse("Hello")`

---

## üîÅ Basic `try-catch`

```csharp
try
{
    int x = 5;
    int y = 0;
    int result = x / y;
}
catch (DivideByZeroException ex)
{
    Console.WriteLine($"Oops: {ex.Message}");
}
```

### üîç How it works

- Code inside `try` is **monitored for errors**
- If an exception occurs, execution jumps to `catch`
- If no error, `catch` is skipped

> ‚ö†Ô∏è Unhandled exceptions crash your app ‚Äî this is **not optional** in real-world projects!

---

## üîÇ Multiple `catch` Blocks

You can catch different types of exceptions:

```csharp
try
{
    File.ReadAllText("data.txt");
}
catch (FileNotFoundException ex)
{
    Console.WriteLine("File missing!");
}
catch (UnauthorizedAccessException ex)
{
    Console.WriteLine("Access denied!");
}
catch (Exception ex)
{
    Console.WriteLine("Something else went wrong.");
}
```

> üß† Always keep `catch (Exception)` **last** ‚Äî it's the most general.

## ‚úÖ `finally` Block (Always Executes)

Used to **clean up resources**, even if an error happens.

```csharp
FileStream file = null;
try
{
    file = File.OpenRead("data.txt");
    // read data
}
catch (Exception ex)
{
    Console.WriteLine("Error: " + ex.Message);
}
finally
{
    file?.Dispose();  // Always runs
}
```

> üîÑ Use `finally` to **close files, release memory, clean up connections**.

## üí• Using `throw`

### 1. Rethrow the same exception

```csharp
catch (Exception ex)
{
    Log(ex);
    throw; // preserve stack trace
}
```

### 2. Throw a new exception

```csharp
if (age < 0)
    throw new ArgumentOutOfRangeException("age", "Age cannot be negative");
```

> ‚úÖ Use `throw` to enforce business rules or signal serious problems

## üßë‚Äçüè≠ Creating Custom Exceptions

Sometimes, built-in exceptions don‚Äôt cut it.

```csharp

{
    public InvalidOrderException(string message) : base(message) { }
}
```

Usage:

```csharp

    throw new InvalidOrderException("Quantity must be greater than zero.");
```

> üéØ Custom exceptions help you **communicate domain-specific problems** clearly.

## üîê Best Practices

| ‚úÖ Do's                               | ‚ùå Don'ts                   |
| ------------------------------------- | --------------------------- |
| Catch specific exceptions             | Catch `Exception` blindly   |
| Use `finally` for cleanup             | Leave resources dangling    |
| Create meaningful custom errors       | Throw `Exception` directly  |
| Rethrow with `throw;` not `throw ex;` | Lose original stack trace   |
| Log errors with context               | Swallow exceptions silently |

## üßØ Real-World Example

```csharp
public string ReadUserData(string filePath)
{
    try
    {
        return File.ReadAllText(filePath);
    }
    catch (IOException ex)
    {
        // Log the file I/O error
        throw new ApplicationException($"Error reading file: {filePath}", ex);
    }
    finally
    {
        Console.WriteLine("Finished attempting to read user data.");
    }
}
```
