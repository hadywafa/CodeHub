# **üÜî Understanding Environment Variables in .NET**

Environment variables in .NET allow you to **configure applications dynamically** without modifying code. They are commonly used for:  
‚úî Setting **configuration values** (e.g., database connection strings, API keys).  
‚úî Controlling **runtime behavior** (e.g., logging level, culture settings).  
‚úî Managing **sensitive information** securely.

---

## **1Ô∏è‚É£ What Are Environment Variables?**

An **environment variable** is a **key-value pair** `stored in the operating system` that applications can `read at runtime`.

- In **Windows**, environment variables are system-wide or user-specific.
- In **Linux/macOS**, they are defined using shell variables.

### **üîπ Example of an Environment Variable**

| Key                      | Value                                                      |
| ------------------------ | ---------------------------------------------------------- |
| `DB_CONNECTION_STRING`   | `Server=mydb.com;Database=mydb;User=admin;Password=secret` |
| `ASPNETCORE_ENVIRONMENT` | `Development`                                              |

Instead of **hardcoding values** in code, .NET applications can **read them dynamically**.

---

## **2Ô∏è‚É£ How to Read Environment Variables in .NET**

.NET provides multiple ways to access environment variables:

### **üîπ Using `Environment.GetEnvironmentVariable` (C#)**

You can retrieve an environment variable in C# like this:

```csharp
using System;

class Program
{
    static void Main()
    {
        string dbConnection = Environment.GetEnvironmentVariable("DB_CONNECTION_STRING");
        Console.WriteLine($"Database Connection: {dbConnection}");
    }
}
```

üìå **If the environment variable does not exist, `GetEnvironmentVariable` returns `null`.**

---

## **3Ô∏è‚É£ Setting Environment Variables**

### **üîπ Windows (CMD & PowerShell)**

#### ‚úÖ **Temporarily (for the current session only)**

```sh
set DB_CONNECTION_STRING="Server=mydb.com;Database=mydb;User=admin;Password=secret"
```

‚úÖ **PowerShell (current session only)**

```powershell
$env:DB_CONNECTION_STRING="Server=mydb.com;Database=mydb;User=admin;Password=secret"
```

‚úÖ **Verify the value**

```sh
echo %DB_CONNECTION_STRING%  # CMD
echo $env:DB_CONNECTION_STRING  # PowerShell
```

#### ‚úÖ **Permanently (System-Wide)**

1. Open **System Properties** ‚Üí **Advanced** ‚Üí **Environment Variables**.
2. Click **New** and enter:
   - **Variable Name:** `DB_CONNECTION_STRING`
   - **Variable Value:** `Server=mydb.com;Database=mydb;User=admin;Password=secret`
3. Restart the terminal to apply changes.

---

### **üîπ Linux/macOS (Terminal)**

#### ‚úÖ **Temporarily (Session Only)**

```sh
export DB_CONNECTION_STRING="Server=mydb.com;Database=mydb;User=admin;Password=secret"
```

#### ‚úÖ **Permanently (All Sessions)**

```sh
echo 'export DB_CONNECTION_STRING="Server=mydb.com;Database=mydb;User=admin;Password=secret"' >> ~/.bashrc
source ~/.bashrc  # Apply changes
```

To check the value:

```sh
echo $DB_CONNECTION_STRING
```

## **4Ô∏è‚É£ Using Environment Variables in `appsettings.json` (ASP.NET Core)**

In **ASP.NET Core**, environment variables can override values in `appsettings.json`.

### **üîπ Example `appsettings.json`**

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=localhost;Database=devdb;User=dev;Password=devpass"
  }
}
```

### **üîπ Override it using an Environment Variable**

Set an environment variable:

```sh
export ConnectionStrings__DefaultConnection="Server=mydb.com;Database=mydb;User=admin;Password=secret"
```

In **ASP.NET Core**, it automatically reads this value at runtime.

### **üîπ How to Read in `Program.cs`**

```csharp
var builder = WebApplication.CreateBuilder(args);
var connectionString = builder.Configuration["ConnectionStrings:DefaultConnection"];
Console.WriteLine($"Database Connection: {connectionString}");
```

üìå **ASP.NET Core automatically maps environment variables to `appsettings.json` using `:` or `__` (double underscore).**

## **5Ô∏è‚É£ Common .NET Environment Variables**

### **üîπ Built-in .NET Environment Variables**

## **6Ô∏è‚É£ Using `.env` Files in .NET**

Instead of setting environment variables manually, you can store them in a `.env` file.

### **üîπ Example `.env` File**

```ini
DB_CONNECTION_STRING=Server=mydb.com;Database=mydb;User=admin;Password=secret
API_KEY=abcdefgh123456
```

### **üîπ Load `.env` in C#**

```csharp
using System;
using System.IO;
using System.Collections.Generic;

class Program
{
    static void Main()
    {
        LoadEnv(".env");
        string dbConnection = Environment.GetEnvironmentVariable("DB_CONNECTION_STRING");
        Console.WriteLine($"Database Connection: {dbConnection}");
    }

    static void LoadEnv(string filePath)
    {
        if (!File.Exists(filePath)) return;
        foreach (var line in File.ReadAllLines(filePath))
        {
            var parts = line.Split('=', 2);
            if (parts.Length == 2)
                Environment.SetEnvironmentVariable(parts[0], parts[1]);
        }
    }
}
```

üìå This function reads the `.env` file and loads its values into **environment variables at runtime**.

## **7Ô∏è‚É£ Best Practices for Using Environment Variables in .NET**

‚úî **DO** use environment variables for sensitive data like API keys and passwords.  
‚úî **DO NOT** commit environment variables to Git (use `.gitignore` to exclude `.env` files).  
‚úî **DO** use `appsettings.json` for general configuration, but allow environment variables to override it.  
‚úî **DO** use `Environment.GetEnvironmentVariable` to access variables at runtime.  
‚úî **DO** use `.env` files for local development but **production should use real environment variables**.
